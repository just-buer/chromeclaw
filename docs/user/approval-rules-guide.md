# 审批规则使用指南

## 1. 概述

**审批规则**（Approval Rules）让你决定 AI 执行哪些操作时需要你手动确认后才能继续，哪些可以直接执行。这是一个"人在环路中"（Human-in-the-Loop）的安全机制，适用于需要精细控制 AI 行为的场景。

典型使用场景：
- 防止 AI 在未经确认的情况下发送邮件、写入文件
- 在 AI 控制浏览器操作时逐步确认每一步
- 对 B 端业务系统（CRM、OA、审核系统）中的关键操作设置强制审批

---

## 2. 在哪里配置

打开扩展的设置页面（Options）→ 左侧导航选择 **Settings** 分组 → 点击 **审批规则**。

---

## 3. 审批层级

ChromeClaw 支持两种互相叠加的审批方式：

### 3.1 静态审批（工具级别）

在 **Settings → Tools** 页面，每个工具行右侧有一个「审批」复选框。勾选后，该工具每次被调用时都会暂停等待你确认。

适合：`gmail_send`、`execute_javascript`、`calendar_delete` 等高风险工具的一键启用。

### 3.2 动态审批规则（参数级别）

在 **Settings → 审批规则** 页面配置，可以根据工具的实际调用参数来决定是否触发审批。

适合：同一个工具在不同参数下需要差异化处理，例如"仅当 AI 搜索词中包含敏感词时才确认"。

**优先级：** 动态规则 > 用户静态配置 > 开发者默认标记

---

## 4. 配置动态审批规则

### 4.1 新建规则

点击右上角 **+ 新建规则**，弹窗分两部分：

#### 第一步：选择场景模板

从 8 个内置场景中点选一个，系统会自动填好工具名称和触发条件：

| 模板 | 触发时机 |
|------|----------|
| 🔍 AI 搜索时 | AI 调用 `web_search` 时 |
| 🖱 AI 点击网页时 | AI 控制浏览器执行点击操作时 |
| 🌐 AI 导航网页时 | AI 控制浏览器跳转 URL 时 |
| 📜 AI 执行 JS 时 | AI 在页面中运行 JavaScript 时 |
| 📧 AI 发邮件时 | AI 调用 Gmail 发送邮件时 |
| 📅 AI 创建日历事件时 | AI 在 Google Calendar 新建事件时 |
| ✏️ AI 写入文件时 | AI 向工作区写入文件时 |
| ⚙️ 自定义规则 | 手动填写所有配置 |

#### 第二步：填写名称和提示消息

- **规则名称**（必填）：用于在列表中识别此规则，可直接使用模板填入的默认名
- **用户提示消息**（选填）：审批弹出时展示给你的说明文字，帮助判断是否允许

点击 **保存** 即完成配置。

### 4.2 高级配置（选填）

展开「高级配置（工具名称 / 触发条件）」可以精细控制：

#### 工具名称匹配

填写工具名称或通配符：

| 示例 | 含义 |
|------|------|
| `web_search` | 精确匹配搜索工具 |
| `browser` | 所有浏览器操作 |
| `gmail_*` | 所有 Gmail 相关工具 |
| `*` | 所有工具 |

#### 触发条件类型

| 条件类型 | 适用场景 | 参数说明 |
|----------|----------|----------|
| **始终触发** | 工具一旦被调用就暂停 | 无需额外参数 |
| **关键词匹配** | 参数值中出现指定词时暂停 | 字段路径 + 关键词列表 |
| **数值阈值** | 参数数值超过设定值时暂停 | 字段路径 + 数值范围 |
| **字段等于** | 某字段的值精确匹配时暂停 | 字段路径 + 期望值 |

#### 字段路径

字段路径对应工具调用时传递的参数名。每个工具有自己的参数结构，高级配置中会显示**可点击的字段提示**：

| 工具 | 常用字段 |
|------|----------|
| `web_search` | `query`（搜索词） |
| `web_fetch` | `url`（目标地址） |
| `browser` | `action`（操作类型：click/navigate/evaluate/type）、`url`、`expression` |
| `write` / `edit` | `path`（文件路径）、`content`（文件内容） |
| `gmail_send` | `to`（收件人）、`subject`（主题）、`body`（正文） |
| `scheduler` | `action`（操作类型：add/remove/update） |
| `execute_javascript` | `code`（JS 代码） |

点击字段提示 badge 可自动填入字段路径，无需手动输入。

## 5. 配置示例

### 示例 1：AI 搜索特定关键词时需确认

**场景**：防止 AI 搜索竞争对手品牌名。

| 字段 | 值 |
|------|-----|
| 工具名称匹配 | `web_search` |
| 条件类型 | 关键词匹配 |
| 字段路径 | `query` |
| 关键词 | `竞品A, 竞品B` |
| 提示消息 | 检测到敏感搜索词，是否允许？ |

### 示例 2：AI 浏览器点击任何按钮时需确认

**场景**：逐步审核 AI 的每一次点击操作（例如 RPA 流程）。

| 字段 | 值 |
|------|-----|
| 工具名称匹配 | `browser` |
| 条件类型 | 字段等于 |
| 字段路径 | `action` |
| 期望值 | `click` |
| 提示消息 | AI 即将点击页面元素，是否允许？ |

> **注意**：当前浏览器工具使用数字 `ref` 编号标识元素（来自快照），无法按元素 ID 过滤特定按钮。若需精准拦截特定元素的点击，可改用「AI 执行 JS 时」模板，并在 `expression` 字段中用关键词匹配元素 ID。

### 示例 3：AI 发送金额超过 1000 的订单时需确认

**场景**：CRM 系统中，超额订单必须人工审核。

| 字段 | 值 |
|------|-----|
| 工具名称匹配 | `crm_create_order` |
| 条件类型 | 数值阈值 |
| 字段路径 | `amount` |
| 大于等于 (gte) | `1000` |
| 提示消息 | 检测到高额订单（金额 ≥ 1000），请确认后继续。 |

### 示例 4：AI 导航到站外 URL 时需确认

| 字段 | 值 |
|------|-----|
| 工具名称匹配 | `browser` |
| 条件类型 | 字段等于 |
| 字段路径 | `action` |
| 期望值 | `navigate` |
| 提示消息 | AI 即将跳转到新页面，是否允许？ |

## 6. 审批弹窗说明

当规则被触发时，聊天界面的工具调用卡片会变为**等待确认**状态：

- 卡片顶部显示黄色 🛡 图标和「Awaiting approval」标签
- 若由动态规则触发，会显示规则名称和提示消息
- 底部有两个按钮：
  - **拒绝**：AI 收到"操作已被用户拒绝"的错误提示，可能会改变策略或报告给用户
  - **同意执行**：继续执行工具，结果正常返回

---

## 7. 规则优先级与执行顺序

- 每条规则有一个**优先级**数字，数字越小越先被评估（默认 100）
- 评估时从小到大遍历所有已启用规则，**第一条命中的规则生效**，后续规则不再评估
- 同一工具既有静态审批标记又有动态规则命中时，动态规则优先（因为动态规则在工具执行前已触发审批请求）

---

## 8. MCP 服务器工具的审批配置

对于远程 MCP 服务器（Remote MCP）上的工具，在 **Settings → MCP** 页面：

- **服务器级别**：勾选「All tools require approval (default)」，该服务器下所有工具默认需要审批
- **工具级别**：在工具列表中单独为每个工具设置审批开关，会覆盖服务器级别的默认值

## 9. 常见问题

**Q：配置了规则但没有触发审批？**

1. 确认规则已启用（列表中复选框为选中状态）
2. 检查「工具名称匹配」是否正确，工具名区分大小写
3. 检查「字段路径」是否匹配实际工具参数名（参考第 4.2 节字段路径表）
4. 如使用关键词匹配，确认关键词在字段值中实际出现

**Q：字段路径填什么？**

字段路径对应工具参数的 JSON key。例如 `web_search` 工具的调用参数是 `{ "query": "搜索词" }`，则字段路径填 `query`。在高级配置中点击对应工具后，系统会自动显示该工具的常用字段 badge 供点击填入。

**Q：可以同时配置多个规则吗？**

可以。多条规则按优先级顺序评估，第一条命中的规则生效。可以为同一工具创建不同条件的多条规则。

**Q：拒绝操作后 AI 会怎么做？**

AI 会收到一条工具错误消息："Tool execution denied by user."。AI 通常会告知用户操作被拒绝，或尝试其他方式完成任务。
